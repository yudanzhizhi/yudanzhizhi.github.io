C语言环境配置
Huang Yue Lv1
  2024-09-19 14:45:14   2024-09-22 22:37:45  

前言

在计算机领域，Linux系统一定是一个绕不开的重要内容。我认为，初学者可以尽早接触Linux，恰好由于C语言的特性，使得它成为一个接触Linux系统的很好的开端。
C语言是一门编译型语言，与之不同的有python等解释型语言。不管在Windows系统还是Linux系统上，都需要安装C语言编译器，才能运行C语言程序。这个时候可能就有疑问了：什么叫编译呢（顾名思义编译器就是编译用的工具）？——编译通俗地说就是翻译，你的电脑实际上是看不懂你所写的代码的，需要一个翻译器将代码转变成电脑能明白的语言，而这个过程就是编译，用到的工具就是编译器。

Windows

对于初学者来说，除了微软家的Visual Studio不推荐，其他都可以用。

不用VS的原因：

  1. 体积大，动辄几十个G，占用巨量磁盘空间
  2. 编译器用的是微软自家的编译器，目前来说我接触到的大部分都是用gcc编译器
  3. VS要求使用安全函数如scanf_s，不然会报错，但是这不是C语言库函数，别人运行代码会找不到这个函数
    PS：上面的第二第三点都可以手动设置，但回想起我当时的情景，动不动就报错，时不时就要配置环境，非常的恼火，直接不用就可以避免很多困扰

Dev-C++

优点：集成了C语言编译器，开箱即用，有简体中文界面
下载地址：https://sourceforge.net/projects/orwelldevcpp/
参考：https://blog.csdn.net/m0_54158068/article/details/124080083
用法：

  1. 进入 SOURCEFORGE 网站，点击 Download 立即下载安装包
  2. 下载好后双击安装包
  3. 暂时不管语言，点击 OK
  4. 点击 I agree
  5. 什么都不用动，点击 Next
  6. 选择安装路径，默认安装在 C 盘，可以自行更改，点击 Install
  7. 等待软件安装，安装完毕后，点击 Finish 。如果取消了勾选 Run Dev-C++5.11 ，则需要自行打开软件，进入下一步软件初始化
  8. 进入到软件初始化界面后，就可以选择语言了，选择完毕后点击 Next
  9. 接着是设置字体颜色图标等，选择完毕后点 Next
  10. 这时候提示 Dev-C++ 已设置成功，再点击 OK 就可以进入到软件里面了

vscode + wsl + Linux（推荐）

vscode

本质上就是一个文本编辑器，类似记事本，但是有很多扩展，以及其他很多功能，所以很多人用。

wsl

Windows Subsystem for Linux，适用于 Linux 的 Windows 子系统，本质上就是一个虚拟机。

Linux

Linux 是一套免费使用和自由传播的类 Unix 操作系统，是一个基于 POSIX 和 UNIX 的多用户、多任务、支持多线程和多 CPU 的操作系统。
Linux 能运行主要的 UNIX 工具软件、应用程序和网络协议。它支持 32 位和 64 位硬件。Linux 继承了 Unix 以网络为核心的设计思想，是一个性能稳定的多用户网络操作系统。

教程

  1. 鼠标右键单击开始按钮，左键单击设置，接着点系统，滑动到最底下，点击可选功能，滑动到最底下，点击更多 Windows 功能，再次滑动到最底下，勾选适用于 Linux 的 Windows 子系统，最后重启一下电脑
  2. 鼠标右键单击开始按钮，左键单击终端，唤出 Windows 命令提示符
  3. 输入 wsl 命令，看到未发现巴拉巴拉的提示
  4. 这时候打开微软商店，也就是 Microsoft Store ，在最上面的搜索框内输入 Ubuntu ，按下回车，下载 Ubuntu 22.04.3 LTS
  5. 下载完毕后，在终端左上角的 + 的右边有一个向下的符号，左键单击，点击 Ubuntu
  6. 这时候会出现一个新的标签页，提示输入一个名字，Please enter a new UNIX username（大概是这样），如果报错 0x800701bc ，就看下面的 24
  7. 输入一个小写字母的名字，简单的就可以，回车，输入一个简单的密码（注意：输入密码的时候没有显示是正常的），输入完毕后按下回车，再次输入刚才的密码
  8. 这时候输入 sudo apt install update ，回车，输入密码，等待命令执行完成
  9. 输入 sudo apt install gcc -y ，等待下载完成
  10. 输入 gcc -v ，出现版本号，说明 gcc 已经安装成功
  11. 输入 mkdir codes ，回车，接着输入 cd codes ，再回车
  12. 输入 vim hello.c ，回车
  13. 按一下键盘上的 i 键，输入 #include<stdio.h> ，回车，输入 int main() { ，回车，输入 printf(“hello, world!\n”); ，回车，输入 return 0;}
  14. 按一下键盘上的 Esc 键，输入 :wq ，回车
  15. 输入 gcc hello,c -o a ，回车，再输入 ./a ，这时候按下回车，就可以看到显示出 hello, world!
  16. 进入 https://code.visualstudio.com/Download ，安装 Windows 版的 vscode
  17. 打开 vscode ，在窗口左边找到一个放大镜图标，以这个放大镜为第二个图标，点击从上往下数的第五个图标
  18. 搜索框中输入 remote ，安装第一个，一般前几个都一起安装了。如果觉得英文界面不习惯，就搜索 Chinese ，下载 Chinese (Simplified) ，按照它的说明操作就可以切换成中文界面
  19. 下载完成后会出现第六个图标，点击，再找到 WSL 下面的 Ubuntu ，点击向右小箭头，就可以连接到虚拟机了
  20. 这时候点击第一个图标，点击打开文件夹，在 vscode 窗口上方居中的位置出现一个 codes ，点击，就可以在左边看到两个文件，一个 a 和一个 hello.c ，这就是刚刚创建出来的文件
  21. 点击 vscode 窗口左上角的查看，点击终端，可以在 vscode 窗口的下方看到一个终端
  22. 输入 gcc hello,c -o a ，回车，再输入 ./a ，回车，就可以运行 hello.c 这个文件了
  23. 解释：gcc 是 C 语言的编译器， hello.c 是你要编译的目标文件的名字，-o 是一个选项，让编译出来的可执行文件改名为你指定的名字， a 就是你指定的名字，所以 gcc hello,c -o a 的意思是用 gcc 编译器将 hello.c 编译出来的可执行文件命名为 a
  24. .表示当前目录，所以 ./a 的意思是运行当前目录下的 a 这个文件
  25. 如果之前在第 5 步的时候遇到报错 0x800701bc ，就鼠标右键单击开始按钮，左键单击终端管理员，点击“是”，打开管理员终端
  26. 接着将 dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart 复制粘贴到管理员终端内，回车
  27. 完成后将 dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart 复制粘贴到管理员终端内，回车
  28. 接着在QQ群里下载 wsl_update_x64.msi 文件，下载完毕后打开，点击 Next ，点击 Finish
  29. 返回刚刚的管理员终端， 输入 wsl –set-default-version 2 ，回车
  30. 这时候从 + 右边向下小箭头打开 Ubuntu 应该就好了

报错

0x800701bc

看上面教程第 24 点

0x80370102

  1. 鼠标右键单击开始按钮，左键单击任务管理器，打开性能页，看虚拟化是否为已启用，如果没启用，就搜索自己的电脑如何进入 BIOS ，在 BIOS 里打开虚拟化
  2. 管理员终端输入 bcdedit /set hypervisorlaunchtype Auto
  3. 重启电脑

sudo apt install update 报错 Unable to locate package update

  1. 输入 sudo apt-get update
  2. 然后就可以 sudo apt install gcc 了
